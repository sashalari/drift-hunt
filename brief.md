# 🎮 БРИФ: Разработка тест-драйва гоночной игры (ПЛАН А — MVP)

## 🎯 ЦЕЛЬ

Создать работающий тест-драйв аркадной гоночной игры за 3 дня, который можно открыть в браузере и сразу играть. Цель: проверить механику, заработок, привлекательность для целевой аудитории (школьники 8-14 лет).

---

## 📊 ТРЕБОВАНИЯ

### Функциональные требования:

**Платформа:**
- HTML5 + JavaScript (один файл index.html)
- Работает в браузере (Chrome, Firefox, Safari)
- Откроется двойным кликом → браузер → можно играть

**Геймплей:**
- Вид сверху (2D, как в спецификации игры)
- Машинка управляется стрелками влево/вправо
- Два уровня сложности (две трассы: Пустыня + Город)
- Макс 2 противника (боты с простым AI)
- Финишная линия определяет победителя
- Машина НЕ ломается при врезании (просто едит дальше)

**Система заработков:**
- 1 место: 1000 монет
- 2 место: 500 монет
- Показывает заработанные монеты после гонки

**Интерфейс:**
- Главное меню (выбор трассы, кнопка ИГРАТЬ)
- Меню паузы (во время гонки)
- Экран результатов (место, монеты)
- Красивое и понятное оформление

### Нефункциональные требования:
- Нет критических багов (можно играть 5+ минут)
- Быстрая загрузка (< 2 сек)
- Адаптивно для разрешения 1280x720 минимум
- Встроенная графика (простые фигуры или спрайты)
- Нет зависимостей на внешние библиотеки (ванильный JS)

### Правила безопасности:
- Нет утечек данных в консоль
- Нет инъекций (все данные валидированы)
- Нет подгрузки внешнего кода
- Локальное хранилище монет (localStorage, не сервер)

## 🛠️ ТЕХНИЧЕСКИЕ ДЕТАЛИ

**Архитектура:**
```
index.html (один файл!)
├── <style> (CSS встроенный)
├── <canvas id="gameCanvas"> (отрисовка)
└── <script>
    ├── Game Manager (управление состоянием)
    ├── Player Car (машинка игрока)
    ├── Bot Car (боты)
    ├── Track Manager (трассы)
    ├── Input Handler (стрелки)
    ├── Physics Engine (движение, столкновения)
    ├── UI Manager (меню, результаты)
    └── Coins System (монеты)
```

**Технологии:**
- Canvas API для отрисовки
- RequestAnimationFrame для игрового цикла
- LocalStorage для сохранения монет
- Ванильный JavaScript (без фреймворков)

**Графика:**
- Спрайты: простые фигуры (прямоугольники, кружки) или нарисованные Левой
- Цвета: яркие, контрастные (для школьников)
- Размер canvas: 1280x720 (или адаптивный)

## 📞 ВЗАИМОДЕЙСТВИЕ КОМАНДЫ

**Разработчик:**
- Пишет код
- Запрашивает спрайты у Дизайнера
- Отправляет на Security + Logic ревью
- Получает замечания → исправляет → повторный ревью

**Дизайнер:**
- Создает спрайты (машины, дороги, интерфейс)
- Дает Разработчику встроить в код
- Получает feedback от Аналитика про UX

**Аналитик:**
- Отслеживает метрики игроки (сложность, баланс)
- Дает рекомендации по UI/UX
- Проверяет что игра понятна целевой аудитории

**Game Expert:**
- Анализирует конкуренцию (Asphalt, NFS, Roblox гонки)
- Дает рекомендации по монетизации
- Проверяет что механика привлекательна для школьников

**Security + Logic:**
- Security: находит уязвимости, утечки
- Logic: находит баги, проверяет соответствие брифу
- Обоих свои замечания → Разработчик исправляет
